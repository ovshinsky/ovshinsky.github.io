用于CPU的芯片级光互连技术

发表于 2025-03-11 阅读数：
本文字数： 4.5k 阅读时长 ≈ 4 分钟

REF. Hao, Qinfen, et al. “A chip-level optical interconnect for CPU.” IEEE Photonics Technology Letters 33.16 (2021): 852-855. 下载

摘要

随着电芯片性能的增长，电信号限制了芯片I/O的功耗、传输距离及信号完整性。在本文中，我们提出了一种新的芯粒架构，用来实现芯片级的光互连。光学I/O芯粒包括一个超小的光收发器和电子元件。分析表明了相比在以太网NIC应用中传统的板级光互连，基于此架构的光互连能够实现1/3的能耗和1/2的面积。通过采用我们提出的此架构，光I/O芯粒可以支持例如CPU、GPU、SWITCH等任意负载。

背景介绍

在芯片上实施光学IO比在板级或者机柜级更有优势。在PCB上的走线会更短，使系统具有更低的能耗，模块尺寸和加工成本。很多文章讨论了芯片级的IO，但很少有文章从系统级的观点出发讨论，例如将光学收发器集成在电芯片上，形成完整的解决方案。
最近，一些完整的芯片级光互连工作已经发表。在[5]及其后续工作中，实现了FPGA芯片的芯片级光互连设计。英特尔[11]开发了一种光收发器芯片，与其交换机芯片集成。这些工作在大多数方面都不同，因此在不同应用之间共享技术具有挑战性，这阻碍了光互连的长期发展。
在本文中，我们提出了一种芯粒架构，能实现片间光互连，以及100Gbps物理层链路，实现一个个超小的光收发器。光IO芯粒可以用于集成任意负载。

架构

如今，ISO/IEC OSI 7层通信协议模型被业界广泛使用。物理层（PHY）分为三个子层：物理编码子层（PCS）、物理介质附加子层（PMA）和物理介质依赖子层（PMD）[1]。当物理介质为光时，PMD将实现为光模块。在板级光互连应用中，PMD与PMA和PCS是分离的。然而，在芯片级光互连应用中，所有组件彼此非常接近，因此它们紧密集成。在[5]和[11]中，PMA与TIA/驱动器集成。在[12]中，CPU设计集成了PMA和TIA/驱动器电路。尽管这些设计提高了集成密度，但它们过于特定，无法为未来芯片级光互连的发展提供通用架构。

芯片级光互连的芯片架构

我们提出了一种通用的芯片级光互连架构，如图1所示。考虑到负载IC（集成电路）对制造工艺敏感，而PMA和PMD不敏感，PMA和PMD被集成为光I/O芯片，负载IC与光I/O芯片之间的连接被标准化以实现解耦开发。因此，负载IC可以从先进的CMOS工艺中受益，而PMA和PMD可以保持在旧工艺上以保持低成本。例如，CPU可以在28nm CMOS工艺中流片，而PMA可以在40nm CMOS工艺中流片。PMD可能使用更旧的工艺或非CMOS工艺。

PCS子层和内部接口的设计

为了使两个芯片独立开发，我们需要定义它们之间的接口。我们选择PCS子层来实现这一目标，因为在以太网等标准中，PCS和PMA之间没有明确的定义，因此更改不会对现有应用产生影响。

由于6.25Gbps接近单端信号的极限，同时也是差分信号的功率效率和成熟度较高的速率，我们选择实现16 x 6.25Gbps的数据速率作为PCS和PMA之间的接口。它将作为负载IC（如CPU）与光I/O芯片之间的接口。如果应用关注低延迟，可以选择使用并行单端信号实现6.25Gbps的I/O。如果不关注低延迟，SerDes是一个不错的选择。

我们设计了一个PCS子层，并将其连接到负载IC的MAC层。PCS子层的内部结构如图2所示。

为了提高PCS的性能，我们采用了多种设计方法。通过实现并行检测，PCS RX可以更快地区分2位同步头的所有可能位置。在每个通道中添加了同步缓冲区以消除时间偏差。实现了I/O缓冲区以动态调整空闲字符，以在插入对齐标记后保持数据速率。在PCS RX中实现了验证模块，以便更早地检测错误，从而减少RX在错误处理上花费的时间。

图3是使用Synopsys公司的VCS软件进行的仿真结果，显示TX和RX可以在9个时钟周期（23.04 ns）和23个时钟周期（58.88 ns）内处理100Gbps的数据。PCS的功耗为42.5207 mW，如图12a所示。

为了测试我们开发的PCS，我们将其与CPU设计一起在28nm CMOS工艺中流片。我们在芯片上添加了一个引脚以测量其带宽。该引脚设计为在MAC帧处理完成后接收脉冲信号。当芯片开始工作时，我们用示波器测量该引脚，并在每500ns间隔内获得四个脉冲。因此，带宽计算如下

其中1512是我们发送的帧的长度（以字节为单位），20是帧头的长度（以字节为单位）。8表示一个字节中有8位，66/64表示我们为每64位添加一个2位同步头，这是以太网协议标准的要求。该公式的计算结果显示，该设计的带宽可以达到101.112Gbps。

光IO芯片的设计

我们实现了光I/O芯片，该芯片由PMA电路和超小型光收发器组成，并通过共封装技术将它们集成在一起。

超小型光收发器

光I/O核心是由日本AIO核心公司设计的硅光子芯片[9]。它是一个集成了驱动器电路、量子点激光器和调制器、光电探测器等其他部件的全合一芯片。由于所有组件都集成在一起，其未封装尺寸仅为5mm x 5mm，与其他部件分离的光收发器相比，尺寸最小。由于它使用量子点激光器，它还可以耐受超过100°C的温度。它还具有插座设计，因此即使在紧密集成中也可以插拔。为了更好地适应数据中心，它采用1310nm波长。因此，它是与电子芯片集成的理想选择。

共封装PMA与光收发器

封装中基板的尺寸为3cm x 2.5cm，厚度为1.49mm。为了减少信号偏差，我们控制每对差分线的内部长度差异在100μm以内。为了同样的原因，四组相邻差分线之间的长度差异控制在200μm以内（如图9a所示）。两个PMA芯片和一个光I/O核心芯片通过共封装集成到一个模块中。图9b是封装模块的图片。图9c是带有两个PMA芯片和超小型光收发器安装槽的基板。

为了隔离不同电源之间的相互干扰，所有电源电压都是独立供电的。因此，电源平面和地平面之间的寄生电感也被最小化，并且快速切换和PMA芯片的电源平面和地平面分别被分成几个独立的部分。

比较

我们比较了芯片级光互连（C-OI）和板级光互连（B-OI）的面积占用和功耗。计算包括电子芯片和光收发器。芯片级光互连包括一个CPU芯片和一个光I/O芯片，光I/O芯片包括PMA和超小型光收发器。相比之下，板级光互连由电子ASIC芯片和可插拔光收发器组成。它们实现了相同的以太网网卡功能。

面积占用的比较

在芯片级光互连实现中，CPU和光I/O芯片在封装前集成，因此可以轻松放入尺寸为3.3cm x 3.3cm的封装中，这是电子芯片封装后的常规尺寸，例如Chelsio公司的T6芯片[10]。同时，典型的板级光互连占用的面积由电子ASIC芯片和可插拔光收发器的总面积决定。对于电子ASIC芯片，我们再次使用3.3cm x 3.3cm = 10.89cm2作为其占用的面积。对于光收发器，四通道小型可插拔（QSFP）光收发器占用的面积为2cm x 6cm = 12cm2。因此，板级光互连占用的面积为10.89cm2 + 12cm2 = 22.89cm2。我们假设电子ASIC芯片和可插拔光收发器之间的走线非常短以简化讨论，这意味着板级光互连实现可能比我们的计算占用更多的面积。

功耗的比较

我们使用Synopsys公司的DC软件计算芯片级光互连的功耗。我们使用的CPU包括4个CPU核心、缓存、总线、MAC和PCS组件。从DC报告（如图12b所示）中，其功耗为1.17W。在III.A节中，我们可以看到PMA的功耗为320mW x 4 = 1.28W。根据AIO的数据表[9]，AIO光收发器的功耗为1.5W。因此，芯片级光互连的总功耗为3.95W。在板级光互连中，典型的100G以太网网卡上的电子ASIC芯片的功耗为12W[10]，可插拔光收发器的功耗为2.5W[7]。因此，总功耗为12W + 2.5W = 14.5W。比较结果如表II所示。

获取商用100G以太网网卡中电子ASIC芯片的组件功耗并不容易，因此我们无法在更深层次上比较芯片级光互连与板级光互连的功耗，以查看哪些部分有助于节能。然而，我们认为有两个因素可能导致我们的芯片级光互连功耗较低。一个是我们设计的PMA芯片在如此短的范围内工作，因此在发送和接收100Gbps数据时消耗的功率较少。另一个是我们选择CPU架构来实现100Gbps以太网数据处理，这显著重用了其现有电路元件，使电子芯片更小并节省了功耗。

结论

我们提出了一种通用的芯片级光互连应用架构，包括CPU芯片和光I/O芯片。我们重新设计了接口以保持两个芯片独立开发。我们基于一对PMA芯片和超小型光收发器通过共封装技术开发了光I/O芯片。我们还在28nm CMOS工艺中流片了带有MAC和PCS的CPU芯片。该架构可以支持任何负载芯片（如CPU、GPU、交换机）实现光I/O。

与板级光互连[7]相比，我们的工作在以太网网卡应用中可以实现1/3的功耗和1/2的面积占用（如表I所示）。通过使用芯片级光互连，以太网网卡可以更小。此外，该工作可以在服务器PCB上实现直接光连接，而无需安装附加的光以太网网卡。
