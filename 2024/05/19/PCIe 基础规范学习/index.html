PCIe 基础规范学习

REF：5.0-1.0-PUB — PCI Express® Base Specification Revision 5.0 Version 1.0

PCIe 简介

PCIe作为第三代通用的I/O互连接口，能实现如下的功能要求：

  1. 兼容多种应用场景（电脑、手机、工作站、服务器、嵌入式系统等等）
  2. 性价比高（低成本高容量）
  3. 支持多平台互连（片间互连、板间互连）
  4. 有多样的物理接口形式（M.2、CEM、U.2、OcuLink等）
  5. 兼容PCI的软件模型（向下兼容）
  6. 性能优秀（低负担 low-overhead、低延迟的通信）
  7. 先进的特性features
    1. 能理解输入的不同类型的数据和指令
    2. 能耗管理和分配机制
    3. 可以支持不同设备
    4. 支持热插拔
    5. 支持link-level和端到端的信号完整性检查
    6. 支持高级的错误报告和处理
    7. 在发射器和接收器能支持不同的共模电压
    8. 容易测试：能够通过简单的连接来测试其电学特性

PCIe的链路（Link）

Link代表两个组件之间的双单工通信通道。PCIe Link的基础组件包括两个低电压差分驱动的信号对：一个发送对和一个接收对。如下图所示。一个PCIe Link包含一个PCIe的物理层。

双单工通信：在每个方向上，数据只能在一个方向上传输（即单向），这与全双工通信（可以同时双向传输数据）有所不同。

PCIe具有如下的基础属性：

  1. ·基本链路：PCIe由两个单向传输差分链路构成，实现发送对和接收对的功能。使用编码的方式嵌入一个信号时钟，从而实现非常高速的数据传输速率。

  2. 信号速率：初始化后，链路必须在支持的信号电平上运行。【不同版本的PCIe协议支持不同信号速率的传输】。

  3. 通道（Lanes）：一个链路（Link）至少需要支持一个通道（Lane)，每个通道代表了一系列差分信号对（一个信号对用于信号发送，一个信号对用于信号接收）。为了扩展带宽，一个链路可能聚合多个通道，用×N来表示，其中N表示通道数量，如×1代表1个通道，×8表示8个通道。

  4. 初始化： 在硬件初始化期间，每个 PCI Express 链路通过链路两端的两个设备进行通道宽度和工作频率的协商来设置。这个过程不涉及固件或操作系统软件。

  5. 对称性：每个链路必须在每个方向上支持相同数量的通道。例如，一个×16链路表示在每个方向上都有16个差分信号对。

PCI Express 到 PCI/PCI-X 桥接器

用于探索、配置和操作的硬件/软件模型

PCIe的硬件/软件模型包括必要的架构，可以用来简单地探索、配置和使用PCIe的功能。模型的核心组件包括：

  1. 软件要能够配置可用的硬件功能

  2. 提供执行基本资源分配的机制，用于可寻址资源，例如内存空间和中断。

  3. 启用/禁用控制功能，接收询问和初始化

  4. 明确定义的顺序和控制流模型，支持硬件/软件接口的一致和鲁棒性

PCIe的配置模型支持两种机制：

  1. PCI兼容的配置机制
  2. PCIe增强的配置机制

每个 PCI Express 链路都通过虚拟的 PCI-to-PCI 桥接器结构进行映射，并与一个逻辑 PCI 总线相关联。这个虚拟的 PCI-to-PCI 桥接器结构可以是 PCI Express 根复合体端口、交换机上行端口或者交换机下行端口的一部分。根端口是一个虚拟的 PCI-to-PCI 桥接器结构，它从一个 PCI Express 根复合体发起一个 PCI Express 层次域。设备被映射到配置空间，以便每个设备都会响应于特定的设备编号。

PCIe分层概览

该文档按照三个独立的逻辑层次来规范架构：事务层（Transaction Layer）、数据链路层（Data Link Layer）和物理层（Physical Layer）。每个层次都分为两个部分：一个处理出站（待传输）信息，一个处理入站（接收）信息，如图1-4所示。这种分层定义的基本目标是为了方便读者理解规范。请注意，这种分层并不意味着特定的PCI Express实现。

PCI Express 使用数据包在组件之间传递信息。数据包在事务层和数据链路层中形成，用于将信息从发送组件传输到接收组件。随着传输的数据包流经其他层次，它们会被扩展以包含其他层次处理数据包所需的额外信息。在接收端，相反的过程发生，数据包从物理层表示转换为数据链路层表示，最终（对于事务层数据包）转换为接收设备的事务层能够处理的形式。 图1-5展示了事务级数据包信息在各层之间的概念流程。
