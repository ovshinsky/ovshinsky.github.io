光互连的研究背景

背景介绍

人工智能是对数据进行分析的技术

近年来互联网用户与智能设备的数量增长迅速，5G通信、超算中心、自动驾驶、移动办公等新技术也不断涌现，社会开始向信息化快速发展，人类已经迈入“大数据”时代。如图1a所示，根据国际数据公司（IDC，International Data Corporation）的预测，到2025年全球将会产生175 ZB的数据量。与此同时，如图1b所示，根据市场研究公司Tractica的一份报告预测，全球人工智能（AI，Artificial Intelligence）产品和服务的年收入将从2016年的6.437亿美元增长到2025年的368亿美元，成为IT领域中增长最快的市场。数据与人工智能发展趋势之间具有强相关并非巧合，这是因为人工智能是一种分析数据的关键技术，通过大量的数据的训练与学习，可以把数据快速转化为新质生产力。

图 1. (a) 国际数据公司（IDC，International Data Corporation）预测的全球预计生成的数据量和（b) Tractica 预测的全球人工智能产品和服务年收入

人工智能和高性能计算应用完成一个指令时需要更多的数据量

当数据中心内部运行的应用发生演化时，数据中心内部对计算、内存、存储、和互连的需求也会随之变化。和传统的工作任务相比，人工智能、机器学习以及其他的高性能运算在每次计算操作中显著地需要使用更多数据量，并且经常需要跨多个节点的并行计算。80%的高性能计算（HPC，High-Performance Computing）和人工智能算法需要大量的数据来完成一个指令，也就是所谓的具有较低的算术强度（AI，Arithmetic Intensity）。

数据带宽是制约计算性能的瓶颈

如图2所示，尽管CPU和GPU在计算和内存带宽上的性能有所不同，但它们性能的屋顶曲线（Rooflines）却相似，在每次浮点运算中，大约只能提供0.1字节的数据（算术强度AI=10）。在高性能计算和人工智能中，如果内存带宽与计算能力不平衡，性能将无法得到充分发挥。如果内存带宽提高到每浮点运算0.5字节（Bytes/FLOP，算术强度AI=2）或更高，预估性能可提升5倍。

图 2. 不同计算硬件平台的屋顶曲线（Roofline）, AI: Arithmetic Intensity, 算数强度。AI = 每秒浮点运算次数（Flops/s) / 每秒数据传输量（Bytes/s）。

AI 和 HPC节点的理想系统内存配置

  1. 内存应能够被所有计算和通信类型的节点直接访问。
  2. 内存系统应具有模块化的设计，能够根据需求进行组合和扩展。
  3. 内存应能够在多个计算节点之间共享和池化，以提高资源利用率。
  4. 内存应具备与 HBM（High Bandwidth Memory） 相似的高带宽特性。
  5. 内存应具备类似 DDR（Double Data Rate） 的容量、延迟和错误校正特性。
  6. 内存应具备类似 LPDDR（Low Power DDR） 的能效特性

计算资源的解聚（池化）使得各个组件能够独立配置和优化，但通常会导致比集成内存更高的延迟。

计算性能随着带宽和内存容量的变化

单实例或多实例计算负载可以用光计算互连（OCI，Optical Computing Interconnect）的内存中【全内存或分层内存配置】进行配置和运行，所有计算核心（CC）都可以被使用。

对于带宽限制的全对全通信、稀疏和大规模问题，性能收益来自于大节点

电学高带宽内存（HBM，High Bandwidth Memory）技术的分析

当下主要用来提高内存带宽的方法是高带宽内存技术（HBM，High Bandwidth Memory）。

HBM技术的能力

对于CPU的运算来说，将CPU从DDR内存迁移到HBM（高带宽内存）可以使CPU接近0.5字节/浮点运算（0.5B/F）的算术强度。对于GPU的运算来说，配备HBM的GPU能实现0.1字节/浮点运算（0.1B/F）的算术强度。

HBM技术的挑战

尽管HBM能提高数据传输速率和带宽，降低计算的算术强度，但当下依然面临着如下三个挑战：

  1. 占用面积：6-8个HBM3堆叠需要占用系统芯片（SoC）上的大量空间（> 600 - 800 mm²）。
  2. 功耗需求：这些HBM3的堆叠消耗约120 - 160 W的功耗。
  3. HBM的成本较高：HBM的成本超过DDR5成本的两倍。
  4. 潜力有限：在实际应用中，进一步提升算术强度的潜力是有限的。

Intel 光计算互连（OCI，Optical Compute Interconnect）的愿景

硅光互连由于其在传输距离（可以在单模光纤上传播数百米），带宽密度（由密集波分复用技术和小光纤间距赋能）上的优势可用于高性能工作负载的计算池优化。

传统方法下带宽密度的扩展依赖于增加波特率和/或封装引脚数量的全电连接，通过在单根光纤上独立调制许多紧密间隔的光波长，密集波分复用（DWDM）提供了更高能效的带宽密度扩展方法。

而传统的可插拔光模块的封装形式（Pluggable Optical Module）不适用于带宽密度扩展的需求，利用这种方式增加带宽密度需要光计算互连（OCI）的封装形式。在这种互连中，硅光收发器与计算芯片共封装，光纤与封装耦合，理想情况下使用可拆卸的光缆。这消除了可插拔封装形式的带宽密度限制，同时也消除了封装与可插拔模块之间高速铜基SerDes接口的需求（以及随之而来的功耗开销），Intel对于光计算互联（OCI）提出了如下的愿景。

图 3.Intel光计算互连（OCI，Optical Compute Interconnect）的方案

  • 超高带宽： ~1Tbps per fiber
  • 远距离传输： 光纤的传输距离可以超过 100 米，远远超过高速信号在传统电缆中的传输距离。
  • 高密度：Shoreline Density > 4x Improvement Over PCIe6
  • 高能效：Energy Efficiency 3 pJ/b ( > 30% Better Than PCIe6)
  • 低延迟：Latency <10 ns + TOF, Comparable to Electrical）
  • 应用场景：计算扩展（Compute Scale-Up），资源池化（Resource Pooling）。

OCI为系统架构带来的新机会

图 3.（a）具有自顶向下开关的典型数据中心网络和（b) OCI使能的扁平化网络结构和池化计算资源

随着人工智能、机器学习模型越来越复杂，规模越来越大，这种计算需求的变化会让我们重新思考数据中心系统的架构方法。

如图3所示，现有系统中两个重要的瓶颈是通过分层网络交换机时计算节点之间的延时和各个计算节点有限的内存。一种优化延时的方法是减少或消除网络交换机，实现扁平化的网络层次结构。在这种方式下，计算节点之间可以通过低延迟的标准（例如 UCIe 或 CXL）直接通信。

通过可用的计算、内存和存储资源的池化，可以改善每个节点的资源可用性和整体资源利用率，具有低延时和灵活互联结构的资源池可以根据特定工作负载需求进行最佳配置和利用。然而，网络的扁平化和资源解聚都需要大带宽、低延时、高能效的互连解决方案，这样的解决方案同样也可以为人工智能、机器学习以及高性能计算的规模扩展提供更大规模的计算集群。

此外，OCI还可以赋能如下的新架构：

  • Dedicated Memory ：为特定的应用或任务专门配置的内存资源
  • Common Node Memory & XPU IO ：在计算节点中共享或统一配置的内存资源
  • Shared/Pooled Memory + IO：多个计算单元可以同时访问相同的内存区域，通过内存池技术动态分配和管理内存资源

在什么情况下OCI比电学互连相比是更好的选择？

AI 和 HPC 的计算、带宽（BW）和扩展要求推动了对可行且经济的集成光学 I/O 技术的需求，当我们需要较大的数据带宽，链接数量，带宽密度和较低的能耗时，OCI将会展现出更好的优势。

从系统视角来看光计算互联的三大价值导向

  1. 性能：当我们扩展计算资源解聚的内存带宽时，人工智能、高性能计算的性能提高，是系统拥有总成本增加速度（TCO，Total Cost of Ownership）的2.5倍。

  1. 可扩展性：电气实现的 AI/HPC内存在扩展过程中受到信号完整性、规模限制、模块化设计和延迟等挑战的制约。集成光学技术通过解锁信号完整性自由度，解决了这些问题，提供了更高的系统扩展能力、更灵活的模块化设计和更低的延迟。此外，集成光学技术还为系统的代际扩展提供了清晰的路径，使得未来的技术发展和系统升级更加可行。
  2. 功耗和成本（Power & Cost）：集成光学技术节省的功耗可以被重新利用，提升系统的计算能力；光学技术在封装方面的成本节省能够直接提高产品的利润率；在相同的系统规模和带宽下，电气互联和光学互联技术具有相似的系统级TCO。

小结

全球数据的快速增长推动了对人工智能（AI）技术的投资，这些技术有助于将数据转化为新质生产力。光学连接的高带宽和合适容量的内存有潜力释放5倍性能，并且可以使性能与成本之比（PERF/TCO）提升2到3倍。

Intel正在开发支持光学计算接口的芯片，以实现高带宽、低延迟和高能效的连接。AI/HPC系统架构对工作负载的敏感性，基于OCI的开发设计参考依然有待研究，用于XPU和光学接口之间协同工作的开放的标准（例如UCIe）依然有待研究，跨供应商的互操作性验证和技术部署的文档工具依然需要完善。

Intel OCI 的物理层实现

Intel的OCI实现方案能够给我们提供技术路径、链路预算和性能指标上的参考，下面我们将进行具体介绍。

异构3D集成芯片是Intel用来实现OCI所需的封装内硅光收发器的一种可行方法，3D集成芯片包括：

  1. 硅光芯片（PIC），包括所有关键的DWDM光学组件（多波长激光器、光调制器和光放大器）

  2. CMOS电子集成电路（EIC）,包括必要的高速和控制电子元件。

EIC和PIC的异构集成使得EIC的能效和性能能够随着CMOS工艺尺度的缩小而提高，光子组件性能的持续提高能随着稳定的硅光平台而发展。可以为未来的技术发展和系统升级提供清晰的路径。

光学链路的整体架构

图 5. 8波DWDM光学发射机（OTX）中的光学链路预算和性能指标

图5展示了八波DWDM强度调制直接检测（IMDD）架构中的光学链路，在O波段光学链路的整体架构中，一个集成的多波长激光器（MWL）用来产生8个光学载波，并将8个光学载波合并送入一组8级级联的微环调制器（MRM），MRM的谐振波长可以通过电调来控制，导致插入损耗（IL）的变化，从而实现光强强度的调制。

MRM的波长选择特性使其能够通过简单级联来实现DWDM（无需显式波长复用器），这是MRM在DWDM系统中对马赫-曾德尔调制器（MZI）的关键优势之一。

宽带半导体光放大器（SOA）能够补偿光路中的损耗，从而实现接收端（RX）信号灵敏度的要求。
接收端级联的微环调制器可以实现光学的波长解复用，从而把不同载波上调制的信号分开。在每个微环的输出端，一个集成的锗硅光电探测器可以把光学信号变成光电流，通过后续的跨组放大器（TIA）进行放大。

发送端和接收端加工了8个微环调制器，它们的半径具有微小的步进（8 nm），微环的半径大约为6 μm左右。由于它们半径的差别，它们的工作频率大约会错开200 GHz（相当于1.15 nm），这样8个环的谐振波长会全部落在一个自由光谱范围（FSR）中。每个环的谐振波长可以通过集成的加热器进行调节。

和用于微环输出光场强度调制的电光效应相比（数十GHz），热光效应的带宽相对来说低很多（~30 kHz）。

在Intel的这个原型机里，Intel通过一个始终打开的闭环温度控制单元（TCU），利用热光效应来维持激光器和微环调制器工作波长的对准，详细说明见后。

此光子集成回路是在Intel的300-mm混合硅光平台上进行的加工。多波长激光器（MWL）和光放大器（SOA）所必须的III-V族的增益介质通过晶圆键合（Wafer-Bonded）的方式加工在硅光芯片剩余的部分上。

接下来，我们展示Intel的光学链路（OTRX）损耗预算，其描述了片上关键O波段光学组件性能，并且解释这些参数如何影响微环调制器（MRM）的驱动要求。

微环调制器（MRM）的性能

图2中TX-PIC里的MRM标称（nominal）半径为6 μm，Q值为4300。

和前面讲的一样，每个微环调制器配备了专用的加热器，能够在工作中随温度变化调整谐振波长，使谐振波长和激光器的出光波长对准。
